# Aether — GDD one-pager (borrador)

## Elevator pitch

Mundo abierto en tercera persona, tono y presentación **anime**, fantasía de **sandbox urbano / aventura** (referencia de diseño: GTA en loop de “explorar → interactuar → progresar”), sin comprometerse aún a sistemas masivos tipo MMO.

## Pilares

1. **Exploración** en mapa contiguo (MVP: un distrito o isla, no una metrópolis entera).
2. **Combate o interacción social** (elegir foco principal para la primera vertical slice).
3. **Identidad visual anime**: personajes, UI, post-procesado coherente con referencias acordadas.

## Core loop (jugador ~90 % del tiempo)

Explorar el entorno → encontrar actividades (misiones secundarias, encuentros, coleccionables) → ganar recursos o historia → desbloquear zonas o capacidades.

*(Iterar con playtests; este párrafo es la hipótesis inicial.)*

## Plataformas objetivo

- **Windows** primero.
- **Linux / macOS** si el pipeline de export en Godot lo permite sin fricción para el equipo.

## MVP (equipo pequeño)

- Menú principal funcional + ajustes básicos (audio / pantalla).
- Un **bioma** jugable, personaje 3rd person, cámara, interacción mínima (NPC o objeto).
- Sin servidores dedicados hasta que el single-player se sienta sólido.

## Fuera de alcance (v1)

- MMO completo, economía global, matchmaking.
- Ciudad procedural gigante sin pipeline de contenido.
